# 05_deliver — Presentation & Delivery (Layer 6)

The output sink for all generated artifacts. This layer holds two things:

1. **`input/`** — staged data flowing between pipeline layers (gitignored; pipeline writes here)
2. **`output/`** — finished artifacts: views, diagrams, manifest (gitignored; generated by `make generate-all`)

The MCP server tools read from `output/` at runtime via the `view_reader` and `manifest` tools. They never write to these folders directly — all writes go through the generate pipeline.

---

## Structure

```
05_deliver/
├── input/                              # Pipeline staging area (all contents gitignored)
│   ├── 01_ingest/                      # jdeps scan outputs + jQA logs + scan markers
│   │   ├── jdeps-output.txt            # (generated) raw jdeps output
│   │   ├── jdeps-jar-edges.csv         # (generated) JAR edge list for Neo4j LOAD CSV
│   │   ├── jqa-scan.log                # (generated) jQA scanner log
│   │   ├── jqa-analyze.log             # (generated) jQA rules engine log
│   │   ├── .jqa-scan-ok                # (generated) scan completion marker
│   │   └── .jqa-analyze-ok             # (generated) analyze completion marker
│   ├── 02_store/                       # Store-layer outputs (schema diffs, migration logs)
│   ├── 03_query/                       # APOC CSV exports from Neo4j
│   │   ├── jar-deps.csv
│   │   ├── top-incoming.csv
│   │   ├── top-outgoing.csv
│   │   └── (empty)_cycles.csv          # Renamed if no cycles found
│   └── 04_generate/                    # Intermediate generation outputs
│
└── output/                             # Finished artifacts (all contents gitignored)
    ├── manifest.json                   # (generated) view index — agent reads this
    ├── views/                          # (generated) Markdown architecture views
    │   └── *.md
    ├── diagrams/                       # (generated) rendered diagram images
    └── *.puml                          # (generated) PlantUML source files
```

## What lands here and when

| Content | Written by | Make target |
|---------|-----------|-------------|
| `input/01_ingest/jdeps-*.` | `01_ingest/jdep/scripts/` | `make jdeps-skip` |
| `input/01_ingest/.jqa-*` | `01_ingest/jqassistant/scripts/` | `make jqa-scan`, `make jqa-analyze` |
| `input/03_query/*.csv` | Neo4j APOC via `neo4j-export-jdeps.cypher` | `make neo4j-export` |
| `output/views/*.md` | `04_generate/generators/scripts/generate_views.py` | `make generate-views` |
| `output/*.puml` | `04_generate/generators/scripts/generate_plantuml.py` | `make generate-diagrams` |
| `output/manifest.json` | `04_generate/generators/scripts/manifest.py` | `make generate-views` |

---

## Important notes

- **All content under `input/` and `output/` is gitignored** — these are pipeline runtime artifacts, not source files
- Each sub-folder contains a `README.md` and `.gitkeep` that ARE committed, to preserve the folder structure in the repo
- The agent interacts with `output/` read-only — views are generated by the pipeline, not by the agent
- To clear all generated output: `make clean`
